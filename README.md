# Evidence-driven-search

Это локальное OSINT-приложение с графическим интерфейсом.

Оно умеет:

- искать по `username`
- искать по `email`
- искать по `phone`
- собирать данные по `domain`
- извлекать `image metadata`
- показывать результаты в виде графа и боковой панели

## Что уже подключено

- `Censys` для доменов
  - сначала пробует `Censys Platform API v3`
  - если его нет, использует `Legacy API` как запасной вариант
- `Search4Faces` для поиска похожих профилей по фото
- `SpiderFoot` как handoff в локальный Web UI
- `Maltego CSV export`
- `Sherlock`, `Maigret`, `Holehe` как опциональные внешние инструменты

## Что нужно для запуска

Минимум:

- Windows
- установленный `Python`
- интернет для внешних источников

Если просто хочешь посмотреть приложение локально, достаточно Python.
Если хочешь, чтобы работали внешние сервисы вроде `Censys` и `Search4Faces`, нужно заполнить ключи в `.env`.

## Как запустить

1. Открой папку проекта `Evidence-driven-search`.
2. Если нужно, заполни файл `.env`.
3. Запусти:

```powershell
./start.bat
```

Можно и так:

```powershell
./start.ps1
```

4. Дождись, пока установятся зависимости.
5. После старта открой в браузере:

```text
http://127.0.0.1:8000
```

## Как заполнить `.env`

В проекте есть файл `.env.example`.

Сделай из него файл `.env` и, если нужно, впиши туда свои ключи.

Основные переменные:

- `OSINT_VERIFY_SSL=false`
- `OSINT_DB_PATH=./osint_graph_app.db`
- `CENSYS_PLATFORM_PAT=...`
- `CENSYS_API_ID=...`
- `CENSYS_API_SECRET=...`
- `SEARCH4FACES_API_KEY=...`
- `SPIDERFOOT_WEBUI_URL=http://127.0.0.1:5001`

Как это работает:

- если есть `CENSYS_PLATFORM_PAT`, приложение использует `Censys Platform`
- если `PAT` нет, но есть `CENSYS_API_ID` и `CENSYS_API_SECRET`, используется `Legacy Censys`
- если ключей нет, приложение всё равно запускается, просто эти интеграции будут отключены

## Что сохраняется в SQLite

Приложение хранит данные в локальной базе `SQLite`.

Там сохраняются:

- кэш поисков
- история запусков
- scan logs
- результаты поисков
- экспортированные `Maltego CSV`

По умолчанию база лежит здесь:

```text
./osint_graph_app.db
```

Если хочешь, путь можно поменять через `OSINT_DB_PATH`.

## Полезные endpoints

Главный UI:

```text
/
```

Проверка состояния:

```text
/api/health
```

История задач:

```text
/api/search/jobs?limit=20
```

Экспорт в Maltego:

```text
/api/export/maltego?type=domain&target=example.com
```

## Если что-то не работает

Проверь по порядку:

1. Установился ли `Python`.
2. Есть ли `.venv` после первого запуска.
3. Заполнен ли `.env`, если ты используешь внешние API.
4. Открывается ли `http://127.0.0.1:8000`.
5. Что показывает `/api/health`.

Если приложение запускается, но внешние результаты пустые, чаще всего причина в том, что не заполнены API-ключи.
